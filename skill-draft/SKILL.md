# Multi-Agent Supervisor Cockpit Skill Draft

## Trigger
Use when a captured source suggests a repeatable agent orchestration UX workflow that needs proof before promotion.

## Steps
1. Pick one real task and write the user, pain, expected output, and success rubric.
2. Run the workflow once with conservative scope; do not invent validation proof.
3. Capture logs, screenshots, diffs, outputs, or review notes as evidence links.
4. Fill the promotion decision card: promote, pilot-only, iterate, or hold.
5. Patch README/prototype/skill claims only when evidence supports them.

## Pitfalls
- Do not turn a launch tweet into a permanent skill before one real trial.
- Keep subjective judgments explicit in the rubric.
- Separate template readiness from validation.

## Verification
- Evidence links exist.
- Decision is grounded in the evidence.
- Claims changed only where the trial supports them.
